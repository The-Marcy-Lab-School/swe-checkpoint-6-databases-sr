# swe-checkpoint-6-databases-sr

Short response questions for the Mod 6 Databases checkpoint. These questions cover authentication, authorization, full-stack architecture, and SQL concepts from across the module.

## Setup

**1. Clone this repo and navigate into it.**

**2. Make a draft branch:**

```sh
git checkout -b draft
```

**3. Write your answers** in `short-response.md` below each question.

**4. When you are done**, add, commit, and push your work. Submit a PR to your instructor for feedback.

---

## Instructions

Answer each question in `short-response.md` in **complete sentences**. Aim for **3–5 sentences per answer** — enough to demonstrate understanding, not just recall.

- Use the exact terms and concepts from the lessons.
- Write in your own words — do not copy definitions verbatim.
- Specific examples and analogies help show how you personally understand the concept.

---

## Grading (27 points)

**Technical Score:** Each of the 8 responses will be evaluated out of **3 points** based on technical accuracy and precision.

| Score | Description |
| ----- | ----------- |
| 3     | Accurate, specific, and uses correct terminology from the lessons |
| 2     | Mostly correct but vague, missing a key detail, or uses imprecise language |
| 1     | Partially correct but contains a significant misconception |
| 0     | Incorrect, off-topic, or missing |

**Writing Quality Score:** The overall submission will receive up to **3 points** for writing quality — grammar, clarity, and sentence structure across all responses.

**Total: 8 questions × 3 pts + 3 writing pts = 27 points**

> **Any submission with typos, grammar mistakes, or other distracting errors will be marked incomplete and must be resubmitted.**
